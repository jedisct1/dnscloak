  • General questions

    What is DNSCrypt?

    DNSCrypt is a protocol that authenticates communications between a DNS client and a DNS resolver. It prevents DNS spoofing. It uses cryptographic signatures to verify that responses originate from the chosen DNS resolver and havenâ€™t been tampered with.

    What is DNS-over-HTTPS (DoH)?

    DNS-over-HTTPS (DoH) is an experimental protocol for performing DNS resolution via the HTTPS protocol. The goal of the method is to increase user privacy and security by preventing eavesdropping and manipulation of DNS data by man-in-the-middle attacks. Another goal of DoH is to improve performance.

    How the app works?

    The app runs a local resolver (dnscrypt-proxy v2) in the background that performs DNS resolutions via DNSCrypt and/or DoH protocols via a user-selected set of resolvers. The app uses a VPN profile to setup the whole system to use local resolver.

    How to use the app?

    Select your preferred resolver(s) and tap start button. First time you will be asked for permission to install VPN profile. After profile install the app will launch a local resolver and setup iOS to use it as a system-wide DNS resolver. You may also enable "Connect On Demand" feature to keep the app running and automatically restarting.

  • Compatibility questions

    Is the app compatible with VPN apps?

    Preamble: all network configurations on the iOS are made using "VPN profiles". There are two main profile types: "enterprise" (listed as "VPN Configurations" in Settings -> VPN, top of the form) and "personal" (listed as "Personal VPN" in Settings -> VPN, bottom of the form). iOS allows only a single profile per type to be active. Enterprise profile superseeds personal profile. Our app is made as a special subtype of enterprise profile. ANSWER: So, if other app's profile is listed as "Personal VPN" one - it is probably compatible, if it is listed in "VPN Configurations" section (DNSCloak's profile listed there) - it is not compatible.

    Is the app compatible with iOS Personal Hotspot mode?

    No. Personal Hotspot launches a local resolver thus causing a port conflict with the app.

  • Resolvers questions

    Does the app support multiple resolvers usage?

    Yes, you can select specific resolvers (or keep your selection empty to use the whole list).

    How to determine fastest resolver(s)?

    App will check RTT for each resolver and will pick candidate based on configurable load balancing strategy. Default strategy is to pick candidate randomly from top two fastest. Other strategies are descibed here. Also, you can enable and view service log - app will print RTT for each used resolver. You can use this infromation to adjust the selection.

    I have selected a resolver provided by famous company, run a synthetic DNS leak test, but it displays my ISP's resolver. Why?

    This is a known situation due to mix of edge cases:

    1. Synthetic DNS leak tests launch large number (hundreds) of simultaneous randomized uncached requests;
    2. Resolvers of large companies do rate limiting, thus failing that requests;
    3. iOS catches that failed responses and fallbacks to system resolver - your ISP's resolver.

    I have IPv4-only connectivity on my home/work WiFi and IPv6-only connectivity on my carrier's network. How to setup your app?

    DNSCloak allows the use of multiple resolvers and checks your connectivity type on network changes. So, you can select both IPv4 and IPv6 resolvers (IPv6 resolvers have "-ipv6" suffix in their names) and the app will switch them automatically.

    How to add a custom list or my own resolver?

    Just add your custom list ("sources") or resolver ("static") via config editor. You can alter or delete default list as well. DNSCloak will present all "sources" and "static"-s found in it's config on the main screen. Also, you may check out the official dnscrypt-proxy's wiki for propper "static" resolver or "sources" list configuration.
